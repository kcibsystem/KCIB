## PROJECT OVERVIEW

Design and build a complete, production-ready web application called **KCIB** (KMITL ChE Inventory & Booking System) — a laboratory equipment and chemical booking system for the Department of Chemical Engineering at King Mongkut's Institute of Technology Ladkrabang (KMITL), Thailand.

The application is a **Single-Page Application (SPA)** using **pure HTML, CSS, and Vanilla JavaScript** (no frameworks). It integrates with Google Apps Script as the backend and Google Sheets as the database. Users authenticate via **Google Sign-In (OAuth 2.0)**.

The design should feel **modern, polished, and professional** — appropriate for an academic institution, with a warm-orange primary accent and clean dark/light contrast. The site must be **fully bilingual (Thai / English)** and **fully responsive (mobile-first)**.

## MULTI-STEP APPROVAL WORKFLOW

### Approval Flow (3 Levels)

```
Student Submits
      ↓
[P1] Advisor Approval   → รอที่ปรึกษาอนุมัติ
      ↓
[P2] Dept. Head Approval → รออนุมัติขั้นที่ 2
      ↓
[P3] Staff Approval      → รออนุมัติขั้นที่ 3
      ↓
✅ อนุมัติแล้ว
```

Rejection at any step → ❌ ปฏิเสธ
User cancels before approval → 🚫 ยกเลิก

### Status Badge Colors
- **รอที่ปรึกษา** → amber background (`rgba(245,158,11,.15)`), amber text
- **รออนุมัติขั้นที่ 2** → blue background (`rgba(59,130,246,.15)`), blue text
- **รออนุมัติขั้นที่ 3** → orange background (`rgba(255,109,56,.15)`), orange text
- **อนุมัติแล้ว** → green background (`rgba(34,197,94,.15)`), green text
- **ปฏิเสธ** → red background (`rgba(239,68,68,.15)`), red text
- **ยกเลิก** → gray background, gray text

## ANIMATIONS & MICRO-INTERACTIONS

### Scroll Reveal
All sections and cards animate in as they enter the viewport:
- `opacity: 0 → 1` + `translateY(24px → 0)`
- Duration: 0.55s, ease-out
- Stagger children: 0.09s delay per item
- Use `IntersectionObserver`

### Loading Splash Screen
On first load, show:
- Full-screen `--dark` overlay
- Animated logo (float + scale)
- Progress bar expanding from 0% to 100% (`--accent` color)
- Disappears after data loads, fades out

### Hero Animations
- Logo: continuous `translateY(-8px → 8px)` float, 3s ease-in-out infinite
- Background orbs: slow opacity/scale pulse (8–12s cycle)
- Particles: random small dots float upward, fade out at top (JS)
- Mouse spotlight: `radial-gradient(circle at var(--mx) var(--my), ...)` tracks cursor

### Card Interactions
- Hover: `translateY(-6px)`, shadow grows, border color → accent
- Active/click: `translateY(-2px)`, slight scale 0.98

### Button Interactions
- Hover: glow `--shadow-accent`, slight upward movement
- Active: scale down `0.96`
- Loading: replace text with spinning ring + "กำลังดำเนินการ..."
- Magnetic CTA: subtle follows cursor within button bounds (JS)

### Modal Animations
- Open: `scale(0.92) translateY(24px) → scale(1) translateY(0)`, cubic-bezier spring
- Close: reverse
- Mobile: `translateY(100%) → translateY(0)` slide up

### Toast Notifications
- Enter: slide in from right (`translateX(120%) → translateX(0)`)
- Exit: fade + slide out
- Auto-dismiss: 4–5 seconds
- Positioning: fixed top-right, stack vertically

### Ripple Effect
On button click: expanding circle from click point, fades out, JS generated

### Stat Counter
Numbers count up from 0 to target value on first viewport entry. Duration: 1.5s

## BOOKING VALIDATION RULES

### Instrument (Timed) Bookings
- Weekdays only (Mon–Fri)
- Operating hours: 09:00 – 16:00
- Max duration: 2 hours per booking
- Advance booking: minimum 3 business days
- No overlap with existing approved bookings for same instrument
- Closed on Thai public holidays (list fetched from Sheets)

### Quantity Bookings (Glassware, Scientific, Chemicals)
- Quantity must be ≥ 1 and ≤ available stock
- Start date must be in the future
- End date must be ≥ start date
- Cannot exceed total inventory

### General
- User must be logged in
- Advisor must be selected
- Course must be selected
- If item already in cart, cannot add duplicate

- No JavaScript frameworks (pure Vanilla JS)
- No CSS preprocessors (pure CSS custom properties)

- Use `IntersectionObserver` for scroll animations (not scroll events)

- Single `index.html`, single `style.css`, single `app.js`
